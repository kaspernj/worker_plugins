Added `WorkerPlugins::DeleteOrphanLinks`. Deletes `worker_plugins_workplace_links` whose target row has been destroyed without the link being cleaned up — a dialect-aware `NOT IN (SELECT ... FROM <target>)` sweep per `resource_type`, reusing `SelectColumnWithTypeCast` for the id-column cast. Consumers schedule it (e.g. from a Sidekiq worker) to keep the links table consistent so that probes like `QueryLinksStatus#checked_count` stay honest without needing an extra JOIN at read time. Links whose `resource_type` doesn't resolve to a Ruby class are left alone; cleaning those up requires human judgement.
