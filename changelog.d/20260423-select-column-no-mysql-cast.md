Broadened `WorkerPlugins::SelectColumnWithTypeCast` to skip the explicit CAST on MySQL / MariaDB whenever neither column is numeric, not just for the `:string`/`:uuid` pair. MySQL's implicit conversion already handles comparisons between any string-ish column types (VARCHAR, CHAR, BINARY, the native UUID type, and types AR doesn't recognize), so casting added cost without buying anything. This removes an unnecessary `CAST(users.id AS CHAR)` from the `WorkerPlugins::RemoveQuery` lookup subquery against `worker_plugins_workplace_links` when `users.id` is a MariaDB native UUID that AR's mysql2 adapter surfaces as an unknown type.
