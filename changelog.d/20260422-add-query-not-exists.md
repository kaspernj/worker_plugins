Rewrote `WorkerPlugins::AddQuery#resources_to_add` to use `NOT EXISTS` with a correlated workplace-links lookup instead of `NOT IN (SELECT ...)`. The previous form nested a `resource_id IN (SELECT CAST(users.id AS CHAR) FROM users)` subquery that triggered a full-table scan of the target model when the outer query was unfiltered (60s+ on ~340k users). The new form uses the `(workplace_id, resource_type, resource_id)` composite index for an index seek per row.
