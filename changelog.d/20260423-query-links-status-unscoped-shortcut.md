Short-circuited the target-model subquery in `WorkerPlugins::QueryLinksStatus#checked_count` when the caller passes an unscoped relation. The previous implementation always ran `SELECT COUNT(*) FROM worker_plugins_workplace_links WHERE workplace_id = ? AND resource_type = ? AND resource_id IN (SELECT DISTINCT <target>.id FROM <target>)`, which on a 340k-row target model took 2+ seconds per check because MySQL materialized every row of the target table. The new form counts directly by `(workplace_id, resource_type)` on the composite index when the query applies no scoping, and keeps the subquery for scoped queries so filter-level semantics survive. Extracted `ApplicationService#relation_unscoped?` and reused it from `RemoveQuery` to keep the detection in one place.
